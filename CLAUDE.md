## Проект

**ZenWidget** — виджет с коллекцией расслабляющих миниигр, живёт поверх других окон. Целевая аудитория: люди, которым нужна микро-пауза во время работы/учёбы. Не геймеры.

**Принципы:** никакого стресса (нет таймеров, game over, очков) · лёгкое по ресурсам · визуально спокойное · каждая игра независима и самодостаточна.

**Стек:** Vanilla JS · Canvas API · CSS Variables · requestAnimationFrame. Никаких фреймворков и внешних зависимостей.

## Платформы

Цель — один исходник JS/Canvas, нативные шеллы под каждую платформу.

| Платформа | Инструмент | Статус |
|---|---|---|
| Mac, Windows | Tauri (Rust + WebView) | 🚧 структура создана, требует `cargo tauri build` |
| iOS, Android | Capacitor (WebView wrapper) | 🚧 конфиг создан, требует `npm install && cap sync` |

**Почему именно так:**
- Tauri — самый лёгкий десктопный шелл (~5 MB), не Electron. Тот же JS/Canvas внутри.
- Capacitor — оборачивает HTML/JS/Canvas в нативное приложение для мобильных без переписывания кода. Canvas API работает через WKWebView (iOS) и WebView (Android) с приемлемой производительностью.
- Flutter/React Native требовали бы полного переписывания — не наш путь.

**Что нужно доработать в коде перед упаковкой в шелл:**

- ~~**Адаптивный canvas**~~ ✅ ResizeObserver + `switchTo()` по дебаунсу, canvas заполняет контейнер
- ~~**Нет фиксированных px-размеров**~~ ✅ Tetris и Snake переведены на вычисляемые размеры от `canvas.width/height`
- **Touch-события** — каждая игра должна полноценно обрабатывать `touchstart / touchmove / touchend` (все текущие игры уже это делают)
- ~~**Виджет поверх окон (десктоп)**~~ ✅ `tauri.conf.json`: `alwaysOnTop: true`, `decorations: false`, `transparent: true`; CSS `.tauri` режим
- **Нет хранилища** — ✅ игры не используют `localStorage` (архитектура уже соблюдает)

## Архитектура

### Интерфейс игрового модуля

```js
export default {
  name:   'bubbles',           // уникальный id (camelCase)
  label:  'Пузыри',            // отображаемое имя
  icon:   '🫧',                // emoji для Game Picker
  init(canvas, ctx, opts),     // инициализация
  update(dt),                  // каждый кадр, dt в мс
  handleInput(event),          // мышь / тач
  pause(),                     // виджет скрыт/неактивен
  resume(),                    // виджет снова виден
  destroy(),                   // очистить listeners, таймеры, ссылки
}
```

### Компоненты ядра

**`core/gameManager.js`** — регистрация игр, переключение (`destroy` старой → `init` новой), главный rAF-loop с throttling (30 FPS активно / 10 FPS в фоне через `visibilitychange`).

**`core/widgetShell.js`** — drag, minimize/expand, Game Picker. Кнопка `⊞` открывает DOM-оверлей поверх canvas (не canvas-рендер) с сеткой emoji-иконок; клик вызывает `gameManager.switchTo(name)`; активная игра выделяется ring/border.

## Соглашения и производительность

- **Нет внешних зависимостей** — только Vanilla JS, никаких npm / CDN
- **Анимация** — только `requestAnimationFrame`, никаких `setInterval` для визуального обновления
- **FPS throttle** — через timestamp из rAF, не через отдельный таймер
- **Memory** — не аллоцировать объекты в update loop; object pool если объектов > 50
- **Canvas** — `clearRect` вместо полного сброса; избегай layout thrashing (не мешай чтение и запись DOM)
- **Listeners** — всегда очищать в `destroy()`
- **Нет `console.log`** в готовом коде

## Ключевые проектные решения

Только нестандартные решения — то, что не очевидно из кода.

**Game Picker — DOM поверх canvas, не canvas-рендер.** Проще стилизовать, нет лишнего рендера иконок. `position: absolute` оверлей поверх canvas-элемента.

**Костёр — петля вовлечённости, не просто частицы.** Drag-and-drop брёвен на кострище → поджечь → брёвна медленно догорают (уменьшаются, темнеют до углей) → огонь угасает. Particle fire привязан к конкретным брёвнам, а не к фиксированной точке на экране.

**Голо-бумага — текстура бейкается один раз в `init()`.** Perlin/simplex шум → hue-mapping → offscreen canvas. Brush управляет только alpha-маской проявки. Одно место = один цвет всегда (как натирание монеты через бумагу). Не пересоздавать текстуру на каждый штрих — это главная ошибка которую нужно избежать.

**Колыбель Ньютона — намеренно завышенная инерция.** Масса шаров завышена чтобы движение ощущалось весомым, не «лёгким».

**Генератор — угловая скорость через EMA разности углов.** `atan2` даёт угол мыши относительно центра ручки; delta angle между событиями делится на dt → мгновенная ω; EMA (α=0.45) сглаживает шум. `power = min(1, |ω| / MAX_OMEGA)`. Фоновые деревья/столбы α→0 при росте скорости, горизонтальные полосы motion-blur нарастают с p>0.45. Дуга на пантографе + искры (particle pool) — при p>0.5.

**Тетрис — нет game over экрана, нет очков.** При заполнении поля вызывается `_newGame()` мгновенно. Вся игра — один файл, без анимации очистки строк. Клавиши на `document`, а не на canvas (canvas не получает фокус).

**Змейка — BFS с тороидальной топологией.** Змейка не управляется игроком: еда ставится кликом, путь находится BFS с обходом сквозь края. При отсутствии пути — случайное блуждание. Голод начинается через 12 с после последнего поедания; каждые 2 с отрезается хвост до минимума 3.

## Как добавить новую игру

1. Создать `games/<name>/index.js` с полным интерфейсом (см. выше)
2. Зарегистрировать в `index.html` через `gameManager.register(...)`
3. Обновить таблицу игр в этом файле
