## Что это за проект

**ZenWidget** — лёгкое десктопное/мобильное приложение-виджет с коллекцией расслабляющих миниигр. Оно живёт поверх других окон, не мешает основной работе и не требует активного внимания. Пользователь может глянуть на него, потыкать пару секунд, и вернуться к своим делам.

**Целевая аудитория:** люди, которым нужна микро-пауза во время работы/учёбы. Не геймеры.

**Ключевые принципы дизайна:**
- Никакого стресса, никакого давления (нет таймеров, нет game over, нет очков если не нужны)
- Максимально лёгкое по ресурсам (CPU/батарея — приоритет)
- Визуально спокойное (мягкие цвета, плавные анимации)
- Каждая миниигра независима и самодостаточна

## Архитектура

### Принцип: каждая миниигра — изолированный модуль

Каждая игра должна экспортировать единый интерфейс:

```js
export default {
  name: 'bubbles',           // уникальное id
  label: 'Пузыри',           // отображаемое имя
  init(canvas, ctx, opts),   // инициализация, получает canvas и context
  update(dt),                // вызывается каждый frame, dt = delta time в мс
  handleInput(event),        // мышь/тач события
  pause(),                   // вызывается когда виджет скрыт/неактивен
  resume(),                  // вызывается когда снова виден
  destroy(),                 // полная очистка (event listeners, таймеры и тд)
}
```

### Game Manager

`core/gameManager.js` отвечает за:
- Регистрацию миниигр
- Переключение между ними (destroy старой → init новой)
- Управление главным animation loop
- Throttling FPS когда виджет не в фокусе (цель: 5-10 FPS в фоне, 30 FPS активно)

### Widget Shell

`core/widgetShell.js` — оболочка:
- Навигация между играми (через Game Picker)
- Drag (перемещение виджета)
- Resize
- Minimize/expand

### Game Picker (меню выбора игры)

Кнопка `⊞` в хедере открывает оверлей поверх canvas с сеткой иконок миниигр.

**Принципы:**
- Оверлей — чистый DOM поверх canvas (не рисуем иконки на canvas)
- Каждая игра имеет поле `icon` (emoji) и `label`
- При клике на иконку: закрыть оверлей → `gameManager.switchTo(name)`
- Кнопки `‹ ›` убрать, заменить одной кнопкой `⊞` (или `☰`)
- Текущая активная игра выделяется в меню (ring/border)

**Интерфейс модуля игры расширяется:**
```js
export default {
  name: 'bubbles',
  label: 'Пузыри',
  icon:  '🫧',          // ← новое поле, emoji для Game Picker
  // ...остальные методы без изменений
}
```

## Соглашения по коду

- **Никаких внешних зависимостей** в браузерном прототипе (без npm, без CDN библиотек)
- **Комментарии на русском или английском** — на усмотрение, но последовательно внутри файла
- **Имена переменных** — camelCase, английские
- **Нет console.log** в готовом коде (только в debug-ветках)
- **requestAnimationFrame** — единственный способ анимации. Никаких setInterval для визуального обновления
- **Throttle FPS:** используй timestamp из rAF чтобы ограничить update до целевого FPS
- **Event listeners** всегда очищаются в `destroy()`

---

## Правила производительности

- Целевой FPS: **30** в активном режиме, **10** в фоне
- **Не аллоцировать объекты в update loop** — создавай заранее, переиспользуй
- **Избегай layout thrashing** — не читай и не пиши DOM в одном цикле
- Частицы и объекты — использовать **object pool** если их больше 50
- Canvas: предпочитай **clearRect** над заменой всего canvas

## Частые вопросы для Claude Code

**Q: Как добавить новую миниигру?**
Создай папку в `/games/`, реализуй интерфейс (init/update/handleInput/pause/resume/destroy), зарегистрируй в gameManager.

**Q: Как тестировать производительность?**
Chrome DevTools → Performance → запись 5 секунд активной игры. Цель: меньше 10% CPU на среднем ноутбуке.

**Q: Можно ли использовать npm-пакеты?**
Нет, в браузерном прототипе — только Vanilla JS.
